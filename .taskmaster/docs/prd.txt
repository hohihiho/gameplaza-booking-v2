# 광주 게임플라자 예약 시스템 PRD (Product Requirements Document)

## 프로젝트 개요
모바일 퍼스트 오락실 예약 시스템으로, 카카오톡 오픈채팅 기반 수동 예약을 웹 기반 자동화 시스템으로 전환하는 프로젝트입니다.

## 핵심 비즈니스 요구사항
- 1인 1대 원칙의 게임기기 예약 시스템
- 24시간 룰: 예약은 24시간 전부터만 가능
- KST 시간대 고정, 익일 새벽 0-5시는 24-29시로 표시
- 영업일 기준 06시 리셋
- 구글 OAuth 인증 (전화번호는 선택사항)
- 관리자 수동 승인 방식

## 기술 스택
- Frontend: Next.js 15, React, TypeScript, Tailwind CSS
- Backend: Supabase (PostgreSQL, Auth, Realtime)
- Deployment: Vercel
- PWA 지원

## 주요 기능 요구사항

### 사용자 기능
1. 회원가입/로그인 (구글 OAuth)
2. 예약 신청 (날짜, 시간, 기기 선택)
3. 예약 상태 확인 및 관리
4. 마이페이지 (프로필, 예약 내역)
5. PWA 푸시 알림

### 관리자 기능
1. 예약 승인/거절 관리
2. 체크인 및 결제 처리
3. 기기 관리 (카테고리-기종-개별기기 3단계)
4. 운영 일정 관리
5. 통계 분석 대시보드
6. 고객 관리

### 시스템 특성
- 모바일 최적화 (99% 모바일 사용자)
- 실시간 동기화 (Supabase Realtime)
- 3G 환경에서도 빠른 로딩
- 다크모드 지원
- 접근성 WCAG 2.1 AA 준수

## 현재 상태
- MVP 기능 90% 완성
- 운영 중인 시스템 개선 작업 진행 중
- 주요 이슈: 태스크 관리 시스템 개선 필요

## 우선순위 작업
1. TDD 기반 전체 리팩토링 (UI 제외)
   - 비즈니스 로직을 도메인 계층으로 분리
   - 테스트 가능한 구조로 재설계
   - 의존성 주입 패턴 적용
   - 기존 UI와 호환되는 API 유지
2. 전화번호 선택사항으로 변경
3. PWA 푸시 알림 구현
4. 예약번호 시스템 구현
5. 성능 최적화
6. 테스트 커버리지 80% 이상 확보

## 제약사항
- 단순함 최우선 (YAGNI 원칙)
- 포트 3000번만 사용
- 한국어 커밋 메시지
- 모바일 퍼스트 설계

## 성공 지표
- 예약 처리 시간 < 3초
- 모바일 로딩 시간 < 2초 (3G)
- 시스템 가용성 > 99.9%
- 사용자 만족도 > 4.5/5
